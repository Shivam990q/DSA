## 🎯 WHAT THIS IS

This is the **third sibling codex**. Where `DSA-Grandmaster-Codex` teaches the *algorithmic mind* and `FULLSTACK-AI-GRANDMASTER-CODEX` teaches the *engineering hands*, this codex teaches the **medium itself**: the programming language.

It is a **civilization-scale knowledge system** for understanding programming languages from **first principles to the frontier** — how they are defined, how they are implemented, why they differ, and how to invent your own. It takes you from *"what is a variable, really?"* to *"here is a working compiler and type checker I built from scratch."*

It answers the questions most engineers never ask but every grandmaster can:

- What *is* a programming language, precisely? (Syntax, semantics, pragmatics.)
- Why does Rust have a borrow checker and Python a garbage collector — and what did each *give up*?
- What actually happens between `source.c` and a running process?
- What is a type, mathematically? Why is `null` "the billion-dollar mistake"?
- How does `async/await` really work? What is a continuation?
- How would I design a new language — and implement it?

## ⚔️ HOW TO USE THIS CODEX

### Path 1: The Foundations Path (recommended)
`00 → 01 → 02 → 03 → 04 → 05`. Build the mental model of what a language *is* and how it runs before touching theory or specific languages.

### Path 2: The Polyglot Path (for working engineers)
`00 → 02 (paradigms) → 07 (language tour)`. Learn the families, then tour eight languages. Best for "I know one language and want to think in many."

### Path 3: The Implementer Path (build a language)
`01 → 05 (compilers) → 08 (semantics) → 10 (design)`. Ends with you writing a lexer, parser, type checker, and interpreter.

### Path 4: The Theorist Path
`01 → 03 → 08`. Lambda calculus, type theory, operational/denotational semantics. The road to reading PL research (POPL, PLDI, ICFP).

### Path 5: The Grandmaster Path
Walk all 13. Emerge able to *learn any language in a weekend*, *read any language's spec*, and *design your own*.
