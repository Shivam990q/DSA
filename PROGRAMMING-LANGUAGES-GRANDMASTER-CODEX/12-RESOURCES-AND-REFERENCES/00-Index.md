## 🧭 The Essential Shortlist

If you read only five things:
1. **Crafting Interpreters** (Bob Nystrom) — build two complete language implementations; the best hands-on PL book, free online.
2. **Structure and Interpretation of Computer Programs (SICP)** — the classic that reshapes how you think about programs.
3. **Types and Programming Languages (TAPL)** (Benjamin Pierce) — the type-theory bible.
4. **Compilers: Principles, Techniques, and Tools** (the "Dragon Book") — the compiler classic.
5. **Programming Language Pragmatics** (Michael Scott) — the best single survey of how languages work.

---

## 📌 Key Takeaways
- Prefer primary sources (specs, canonical books) over summaries.
- Build things (Crafting Interpreters) alongside reading theory (TAPL, SICP).
- The classics compound — they're old because the ideas are permanent.
