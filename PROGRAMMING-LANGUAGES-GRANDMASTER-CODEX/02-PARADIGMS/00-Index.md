# 🌌 Paradigms — The Great Families of Thought

> *"A paradigm is not a feature set. It is a way of decomposing reality into a program."*

A **programming paradigm** is a fundamental style of structuring computation — a different answer to "what is a program *made of*?" Objects? Functions? Facts and rules? Each paradigm rewires how you decompose problems. Learning a new paradigm is the single fastest way to become a better programmer, even in languages you already know.

## 🧭 The Paradigms at a Glance

| Paradigm | A program is... | Key idea | Exemplars |
|----------|-----------------|----------|-----------|
| **Imperative** | a list of commands | change state step by step | C, assembly |
| **Procedural** | commands grouped into procedures | decompose into reusable routines | C, Pascal |
| **Object-oriented** | objects sending messages | bundle data + behavior; hide state | Java, C++, Smalltalk |
| **Functional** | composed pure functions | avoid mutation; values in, values out | Haskell, Lisp, Elm |
| **Logic** | facts + rules | declare truths; let the engine search | Prolog, Datalog |
| **Declarative** | a description of the result | say *what*, not *how* | SQL, HTML, regex |
| **Concurrent** | cooperating processes | many things at once | Erlang, Go |
| **Reactive** | reactions to event streams | data flows, propagate changes | RxJS, Elm |

---

## 🌍 The Big Truth: Most Languages Are Multi-Paradigm

Pure paradigms are rare. Modern languages blend:
- **Python, JavaScript, Scala, Rust, Kotlin** are all multi-paradigm — imperative + OO + functional.
- The paradigm is a *style you choose within the language*, not a cage.

Learn the paradigms as *lenses*. A great engineer switches lens to fit the problem: functional for data pipelines, OO for domain modeling, imperative for tight loops, declarative for queries.

---

## 📌 Key Takeaways
- A paradigm answers "what is a program *made of*?" — commands, objects, functions, facts, or streams.
- Learning a new paradigm improves your code in *every* language.
- Most modern languages are multi-paradigm; treat paradigms as interchangeable lenses.
