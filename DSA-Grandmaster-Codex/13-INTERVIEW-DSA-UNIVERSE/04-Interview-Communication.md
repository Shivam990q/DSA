# 🗣️ Interview Communication

> *"The interviewer can't read your mind. If you don't say it, it didn't happen."*

---

## I. WHY COMMUNICATION IS HALF THE INTERVIEW
At FAANG, two candidates who both solve the problem are differentiated by communication. The interviewer evaluates HOW you think, not just the final answer.

---

## II. THE NARRATION STRUCTURE
```
1. UNDERSTAND
   "Let me restate to confirm I understand..."
   "Can I assume the input is sorted? Non-empty?"

2. EXAMPLES
   "Let me trace the given example..."
   "What about edge cases — empty, single element?"

3. APPROACHES
   "The brute force would be O(n²) because..."
   "Can we do better? If we use a hash map..."

4. DESIGN
   "I'll go with the hash map approach. My plan:
    Step 1... Step 2..."

5. CODE
   (narrate each significant line's purpose while typing)

6. TEST
   "Let me trace example 1... now an edge case..."

7. COMPLEXITY
   "Time: O(n). Space: O(n). Alternative with O(1) space..."
```

---

## III. THE DOs
- Think aloud constantly
- Ask clarifying questions early
- State assumptions explicitly
- Acknowledge tradeoffs
- Accept hints gracefully ("Good point, let me reconsider...")
- Test before declaring done

---

## IV. THE DON'Ts
- Go silent for 5+ minutes
- Code before sharing approach
- Argue with the interviewer
- Pretend to understand when you don't
- Get defensive about bugs
- Ignore hints (they're trying to help you pass)

---

## V. HANDLING "I'M STUCK"
Don't go silent. Say:
- "Let me think about what's making this hard..."
- "The brute force is X; the bottleneck is Y; I'm looking for a way to avoid Y..."
- "Could I get a small hint on the direction?"

Verbalizing your stuck-ness often unsticks you AND shows your reasoning.

---

## VI. HANDLING HINTS
Interviewers drop hints to help you pass. Embrace them:
- "Oh, that's a great point. So if I sort first..."
- Don't be stubborn. Adapt quickly.

---

## VII. THE COMMUNICATION RUBRIC (self-evaluate via recording)
```
☐ Restated the problem
☐ Asked clarifying questions
☐ Shared approach before coding
☐ Narrated while coding
☐ Tested with examples + edge cases
☐ Stated complexity
☐ No long silences
☐ Responded well to hints
```

---

## VIII. PRACTICE
- Record yourself solving (watch for silence, fumbling)
- Mock interviews with strangers ([Pramp](https://www.pramp.com), [interviewing.io](https://interviewing.io))
- Explain solutions to a non-technical friend
