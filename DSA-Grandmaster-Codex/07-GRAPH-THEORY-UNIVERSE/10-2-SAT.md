# 🌐 2-SAT

> *"Boolean constraints with two variables per clause — solvable in linear time via SCC."*

---

## I. THE PROBLEM
Given boolean variables x₁...xₙ and clauses each of the form (a ∨ b) where a, b are literals (a variable or its negation), find an assignment satisfying ALL clauses — or report impossible.

General SAT is NP-complete, but **2-SAT (exactly 2 literals per clause) is polynomial** — in fact linear.

---

## II. THE IMPLICATION GRAPH ⭐
Each clause (a ∨ b) is equivalent to two implications:
- ¬a → b   (if a is false, b must be true)
- ¬b → a

Build a directed graph with 2n nodes (each variable + its negation). Add both implication edges per clause.

---

## III. THE SCC TRICK ⭐
Compute SCCs of the implication graph (Tarjan/Kosaraju).
- **Unsatisfiable** iff some variable x and its negation ¬x lie in the **same SCC** (x ⟹ ¬x ⟹ x is contradictory).
- **Satisfiable** otherwise. To build an assignment: in the condensation DAG (reverse topological order), set x = true if x's SCC comes after ¬x's SCC (i.e., compare SCC order of x and ¬x).

O(V + E) = O(n + clauses).

---

## IV. MODELING CONSTRAINTS AS 2-SAT
Many constraints reduce to 2-SAT clauses:
- "at least one of a, b" → (a ∨ b)
- "a implies b" → (¬a ∨ b)
- "exactly one of a, b" → (a ∨ b) ∧ (¬a ∨ ¬b)
- "a and b can't both be true" → (¬a ∨ ¬b)
- Forcing a variable true → (a ∨ a)

The art is translating the problem's conditions into 2-literal clauses.

---

## V. APPLICATIONS
- Constraint satisfaction with binary choices
- Scheduling/assignment with pairwise conflicts
- "Choose one of two options per item such that all pairwise constraints hold"
- Map labeling, circuit design, puzzle solving

---

## VI. COMPLEXITY
O(n + m) where m = number of clauses (each adds 2 edges). Linear — very efficient.

---

## VII. PROBLEMS
- [CSES](https://cses.fi/problemset/) "Giant Pizza" ⭐ (classic 2-SAT)
- CF problems tagged "2-sat"
- Many ICPC constructive problems reduce to 2-SAT
- Boolean assignment / conflict problems
