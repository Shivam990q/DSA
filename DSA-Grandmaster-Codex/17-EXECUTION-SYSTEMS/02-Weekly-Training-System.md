# 📅 Weekly Training System

> *"What you do every day matters. What you do every week defines arc."*

---

## I. THE WEEKLY ARC

A week is the smallest unit where you can:
- Cover variety (multiple topics)
- Test consistency (5-7 days)
- Reflect with perspective

Plan in weeks. Track in weeks. Adjust in weeks.

---

## II. THE STANDARD WEEK

```
Mon — New topic intro      (3h)
Tue — Topic deepening      (3h)
Wed — Mixed practice       (2-3h)
Thu — Hard problem day     (2-3h, single problem)
Fri — Live contest         (2-3h + 1h analysis)
Sat — Upsolve + mock       (3-4h)
Sun — Reflection + plan    (1-2h light)
                            ────
                            17-21h/week
```

---

## III. THE LIGHT WEEK (10 hours)

For working professionals, students with busy semesters:

```
Mon — 1 hr (warmup + 1 medium)
Tue — 1 hr (theory + 1 medium)
Wed — 1.5 hr (2-3 mediums)
Thu — 1.5 hr (1 hard problem)
Fri — 2 hr (Codeforces Div 3)
Sat — 2 hr (upsolve + mock)
Sun — 1 hr (review + plan)
```

---

## IV. THE INTENSIVE WEEK (35-40 hours)

For full-time CP/interview prep:

```
Mon — 6h (theory 1.5h + 5 problems)
Tue — 6h (topical drill: 8-10 problems)
Wed — 6h (mixed: 6-8 problems)
Thu — 6h (1-2 hard problems + reflection)
Fri — 6h (live contest + immediate upsolve)
Sat — 6h (full upsolve + 2 mocks)
Sun — 4h (re-implement key algos + plan next week)
```

---

## V. THE TOPICAL IMMERSION WEEK

Every 4-6 weeks, dedicate ALL practice to ONE weak topic.

```
Mon-Sun: ALL problems on [DP / Graphs / Strings / etc.]
30-50 problems total
Theory study daily (1 chapter)
Implement core algorithms from scratch
End-of-week: blog post / notes
```

This is the fastest way to break a plateau.

---

## VI. THE WEEKLY METRICS

Track every Sunday:

| Metric                     | Target              |
|----------------------------|---------------------|
| Problems solved            | 15-30               |
| Hours of deep practice     | 10-25               |
| Editorials read            | 5+                  |
| New patterns identified    | 2-5                 |
| Mistakes logged            | 5-10                |
| Live contests participated | 1-2                 |
| Mock interviews            | 1-2                 |
| Re-implementations         | 1-3 algorithms      |

---

## VII. SUNDAY RITUAL (60-90 min)

```
Step 1: Review week's solved problems (15 min)
   - Re-read your reflection notes
   - Spot patterns

Step 2: Update mistake log (10 min)
   - Categorize: bug type, root cause
   - Look for recurring categories

Step 3: Re-implement 2-3 key algorithms (30 min)
   - From memory, on whiteboard or fresh editor

Step 4: Plan next week (15 min)
   - Pick topic for Monday
   - Block time
   - Set 1 weekly goal (e.g., "solve 5 segtree problems")

Step 5: Light review (10 min)
   - Skim cheatsheets
   - Stretch, hydrate
```

---

## VIII. THE QUARTERLY RHYTHM (12 weeks = 1 quarter)

```
Weeks 1-4: Foundation reinforcement (cover gaps in basics)
Weeks 5-8: Frontier expansion (advanced topics)
Weeks 9-11: Contest-heavy (4-5 contests/week)
Week 12: Review + reflection + reset
```

Quarter 1 of the year: foundational mastery.
Quarter 2: depth in one specialty.
Quarter 3: contest grinding.
Quarter 4: integration + interview prep + applying.

---

## IX. THE 90-DAY MILESTONE

Every 90 days, ask:

1. **Knowledge added**: Which topics did I master?
2. **Rating delta**: CF / AC / LC progression?
3. **Mistake categories**: Which bugs am I no longer making?
4. **Identity shift**: Do I think like a stronger version of myself?

Adjust the next 90 days based on answers.

---

## X. THE BURNOUT GUARDRAIL

If you start dreading practice → you're burning out.

Signs:
- Skipping multiple days in a row
- Quality of practice drops (zombie mode)
- Disengagement from problems
- Procrastination via Discord/forums

Action:
- 1 full week off (no problems, no theory)
- Read fiction, exercise, see friends
- Return at 50% volume; ramp slowly

**Burnout is reversible. Quitting permanently is not.** Take rest seriously.

---

## XI. ACCOUNTABILITY MECHANISMS

- **Public commit**: tweet or blog your weekly goal
- **Buddy system**: weekly call with another person on the path
- **Streak tracker**: GitHub-style green squares
- **Visible scoreboard**: rating graph posted on desk

Public + visible → harder to skip.
