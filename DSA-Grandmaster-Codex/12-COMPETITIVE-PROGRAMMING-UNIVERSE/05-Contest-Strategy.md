# ⚔️ Contest Strategy

> *"You win contests in the months before. But tactics on the day still matter."*

---

## I. PRE-CONTEST
- 24h: light practice, sleep 8h
- 2h: eat, hydrate
- 30 min: warmup problem, open template, quiet space, deep breaths

---

## II. THE OPENING
- Read ALL problems first (5-10 min)
- Sort by perceived difficulty
- Note constraints for each (algorithm class hints)

---

## III. PROBLEM ORDER
- Solve easiest first (bank points + confidence)
- Don't fixate on problem A if it's tricky — sometimes B is easier
- For ICPC/IOI: grab easy subtasks across all problems first

---

## IV. THE SWITCHING RULE
- 30-40 min stuck with no progress → SWITCH problems
- Return later with fresh eyes
- Sunk-cost fallacy kills contest performance

---

## V. PER-PROBLEM PROTOCOL
1. Understand (read 2-3×, restate)
2. Constraints → algorithm class
3. Brute force complexity check
4. Optimize / recognize pattern
5. Convince yourself of correctness
6. Code cleanly
7. Test on examples + edge cases
8. Submit

---

## VI. THE ENDGAME (last 10-15 min)
- Verify submitted solutions aren't pretests-only risk
- DO NOT submit untested code in panic
- Stress-test uncertain solutions if time permits

---

## VII. EMOTIONAL REGULATION
- WA happens — it's information, not failure
- Don't rage-submit
- Don't tilt after a bad start
- Breathe; return to the protocol

---

## VIII. THE TYPE-SPECIFIC PLAYBOOKS
- **CF Div 2**: A-B fast, C deep, D if time
- **[AtCoder](https://atcoder.jp) ABC**: A-D fast, E is the mover
- **ICPC**: team coordination, easy trio first
- **IOI**: subtask maximization

---

## IX. POST-CONTEST
- Upsolve EVERYTHING (the real learning)
- Read editorials of unsolved
- Log patterns + mistakes
