# 📄 Resume & Profile Building

> *"The best DSA skills are useless if your resume never gets you the interview."*

---

## THE RESUME RULES
1. **One page** (for students/early career; 2 pages max for senior)
2. **Clean, simple format** — no fancy graphics (ATS systems may choke on them)
3. **Impact over duties** — "Reduced API latency by 40%" not "Worked on APIs"
4. **Quantify everything** — numbers, metrics, scale
5. **Tailor to the role** — keywords from the job description
6. **Proofread** — typos signal carelessness
7. **PDF format** (preserves formatting)

---

## THE STRUCTURE
```
NAME | email | phone | LinkedIn | GitHub | LeetCode/Portfolio

EDUCATION
  Degree, Institution, CGPA, graduation year

SKILLS
  Languages: C++, Python, Java, ...
  Technologies: React, Node, SQL, AWS, ...
  CS: DSA, OS, DBMS, CN, OOP, System Design

EXPERIENCE (if any — internships, jobs)
  Role, Company, Dates
  • Impact bullet (action verb + what + quantified result)
  • ...

PROJECTS ⭐ (most important for students)
  Project Name | Tech stack | [GitHub/Live link]
  • What it does + your contribution + impact/scale
  • ...

ACHIEVEMENTS
  • Codeforces rating, LeetCode count, hackathon wins, ranks

(Optional) POSITIONS OF RESPONSIBILITY / CERTIFICATIONS
```

---

## THE IMPACT BULLET FORMULA
```
[Action verb] + [what you built/did] + [technologies] + [quantified result]
```
Examples:
- "Built a real-time chat app using React, Node.js, and WebSockets, supporting 1000+ concurrent users"
- "Optimized database queries with indexing, reducing average response time from 800ms to 120ms"
- "Implemented an LRU cache in C++, improving lookup performance by 60%"

---

## PROJECTS — THE STUDENT'S SECRET WEAPON
Without much experience, projects ARE your experience:
- **2-3 strong projects** > 10 weak ones
- Each should: solve a real problem, be deployed/demoable, have clean code + README
- Show range: a full-stack app, an algorithmic project, maybe an OSS contribution
- Be ready to discuss every line and tradeoff in interviews

### Project ideas by interest
- **Web**: full-stack app (task manager, social app, e-commerce)
- **Systems**: build a database, key-value store, web server from scratch
- **Algorithms**: a visualizer, a search engine, a recommendation system
- **ML/Data**: a model + pipeline (if relevant)
- **OSS**: contribute to a real open-source project

---

## THE ATS REALITY
Many companies use Applicant Tracking Systems that parse resumes:
- Use standard section headings
- Include keywords from the job description
- Avoid tables/graphics/columns that confuse parsers
- Use a clean, single-column format

---

## ONLINE PROFILE CHECKLIST
- [ ] **LinkedIn**: complete, professional photo, projects, activity
- [ ] **GitHub**: pinned projects, good READMEs, clean commit history
- [ ] **[LeetCode](https://leetcode.com)**: solid solved count (signals consistency)
- [ ] **[Codeforces](https://codeforces.com)** (for CP companies): rating
- [ ] **Portfolio site** (optional): showcases projects
- [ ] **Resume**: tailored, clean, PDF

## THE COMMON MISTAKES
- Listing duties instead of impact
- No quantification
- Too long / cluttered
- Generic (not tailored)
- Typos
- Weak/no projects
- Fancy formatting that breaks ATS

---

## THE PRINCIPLE
Your resume's ONE job: get you the interview. Make it clean, impactful, quantified, and tailored. Then your prepared skills do the rest.
