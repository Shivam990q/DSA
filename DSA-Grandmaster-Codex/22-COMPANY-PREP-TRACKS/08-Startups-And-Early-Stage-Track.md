# 🚀 Startups & Early-Stage Track

> Seed, Series-A/B startups, YC companies, early-stage product teams.

A DIFFERENT game: less DSA puzzle-grinding, more "can you BUILD and SHIP?" Your projects, practical skills, and versatility matter most.

---

## THE HIRING FUNNEL (varies widely)
```
1. Resume / referral / direct application (Wellfound, YC jobs)
2. Founder/engineer screen (often a conversation about your projects)
3. Practical round:
   - Take-home assignment (build a feature/app) OR
   - Live coding (practical, not pure DSA) OR
   - Pair programming
4. Culture/founder fit
5. Offer (often equity-heavy)
```

---

## WHAT THEY TEST
- **Can you build?** ⭐ — real, working software, end-to-end
- **Practical coding** — debugging, APIs, frameworks, databases
- **Versatility** — full-stack ability, learning speed
- **Ownership** — startups need self-starters
- **Culture fit** — small teams, high trust
- **Moderate DSA** (some do [LeetCode](https://leetcode.com)-style, many don't)

---

## WHAT MATTERS MOST
1. **Projects / portfolio** ⭐⭐ — a working app, a GitHub with real code
2. **Practical skills** — your tech stack (React, Node, Python, databases, cloud)
3. **Shipping ability** — have you built and deployed something real?
4. **Communication + ownership**
5. **DSA** (secondary, but basics still expected)

---

## THE PREP PLAN (project-focused)
```
Build 2-3 real, deployed projects:
  - A full-stack app (frontend + backend + database + deployed)
  - Something that solves a real problem
  - Clean code, good README, live demo

Plus:
  - Basic-to-medium DSA (for any coding screens)
  - Know your stack DEEPLY
  - Be ready to discuss tradeoffs in your projects
```

---

## THE RESOURCE STACK
- **Build**: pick a stack (e.g., MERN, Next.js + Postgres, Django) and build
- **Deploy**: Vercel, Netlify, Railway, AWS/GCP free tier
- **DSA basics**: [NeetCode](https://neetcode.io) 75 / Blind 75 (for coding screens)
- **System design lite**: how your app scales
- **Find jobs**: [Wellfound](https://wellfound.com) (AngelList), YC's Work at a Startup, Twitter/X

---

## THE BAR
- A portfolio that proves you can build + ship
- Practical coding competence
- Learning agility (startups change fast)
- Basic DSA

---

## THE STRATEGY
- Startups care about **what you've built**, not your LeetCode count.
- A strong **GitHub + deployed projects + a good story** beats a medium LeetCode profile here.
- **Equity** is part of comp — understand it (could be huge or worthless).
- Higher risk, higher potential reward, faster learning, more ownership.
- Great for those who want to learn fast and wear many hats.

---

## THE TRADEOFFS
| Pros | Cons |
|------|------|
| Fast learning, ownership | Less job security |
| Equity upside | Lower base (sometimes) |
| Broad experience | Less mentorship/structure |
| Direct impact | Longer hours (sometimes) |
