# ✍️ Whiteboard Lab — Solving Without an IDE

> *"No autocomplete. No compiler. No run button. Just you, the marker, and the problem."*

---

## I. WHY WHITEBOARD?
Many interviews (especially onsites) use a whiteboard or a plain doc with no syntax highlighting, no autocomplete, no execution. This is a DIFFERENT skill.

---

## II. WHAT BREAKS WITHOUT AN IDE
- No autocomplete → must remember exact API names
- No compiler → syntax errors go uncaught
- No run → can't test by executing
- No backspace luxury → planning matters more
- Limited space → must write compactly

---

## III. THE WHITEBOARD LAYOUT
```
┌─────────────────────────────────────────┐
│ [Top-left]          [Top-right]          │
│ Assumptions         Complexity           │
│ Examples                                 │
├─────────────────────────────────────────┤
│ [Center]                                 │
│ Main code (write LARGE, leave space)     │
│                                          │
├─────────────────────────────────────────┤
│ [Bottom]                                 │
│ Trace examples / edge cases              │
└─────────────────────────────────────────┘
```

---

## IV. THE WHITEBOARD PROTOCOL
1. **Plan before writing** (you can't easily insert lines later)
2. **Pseudocode first**, then transcribe to real code
3. **Write LARGE** with space between lines (for corrections)
4. **Leave blank lines** to insert forgotten code
5. **Trace by hand** to test (no compiler)
6. **Cross out**, don't erase frantically

---

## V. THE PRACTICE
- Solve 1 problem/day on actual paper or whiteboard
- No IDE, no autocomplete
- Write FULL syntactically-correct code
- Then type it into a compiler — count the syntax errors
- Goal: drive syntax errors to zero

---

## VI. MEMORIZE YOUR LANGUAGE'S CORE API
Without autocomplete, you must KNOW:
- C++: `sort`, `lower_bound`, `priority_queue`, `unordered_map`, iterators
- Python: `sorted`, `heapq`, `collections`, `bisect`, comprehensions
- Java: `Arrays.sort`, `PriorityQueue`, `HashMap`, `Collections`

Drill these until automatic.

---

## VII. EVALUATION
```
☐ Planned before writing
☐ Code is syntactically correct (verify in compiler after)
☐ Handled edge cases
☐ Traced an example by hand
☐ Wrote legibly with room for edits
☐ Stated complexity
```

---

## VIII. THE PAYOFF
Whiteboard practice makes IDE coding feel easy. If you can solve on paper, you can solve anywhere.
