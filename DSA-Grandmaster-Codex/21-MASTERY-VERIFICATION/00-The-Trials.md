# 🏆 The Trials — Mastery Verification

> *"Knowing is not mastery. Performing under pressure is mastery."*

---

## 📜 THE THREE TRIALS

To claim mastery of any level, you must pass three trials:

### 🥉 Trial 1: **The Implementation Trial** (Mechanical Mastery)
Implement specified algorithms from scratch, in <X minutes, no references.

### 🥈 Trial 2: **The Problem Trial** (Application Mastery)
Solve specified problems from scratch, in time, with correct edge handling.

### 🥇 Trial 3: **The Teaching Trial** (Conceptual Mastery)
Explain the concept to a confused beginner. They must understand.

If all three pass → mastery is verified.

---

## 🪜 THE TRIALS BY LEVEL

### Level 0 (Programming Foundations) — The Genesis Trial
- **Trial 1**: in 60 min, write: factorial, GCD, palindrome check, reverse string, fibonacci.
- **Trial 2**: solve [LC 1: Two Sum](https://leetcode.com/problems/two-sum), [LC 9: Palindrome Number](https://leetcode.com/problems/palindrome-number).
- **Trial 3**: explain "what is recursion?" to a non-programmer.

### Level 1 (Beginner DSA) — The First Steps Trial
- **Trial 1**: implement binary search, Kadane's, BFS, DFS in <40 minutes total.
- **Trial 2**: solve [LC 53](https://leetcode.com/problems/maximum-subarray), [LC 141](https://leetcode.com/problems/linked-list-cycle), [LC 20](https://leetcode.com/problems/valid-parentheses).
- **Trial 3**: explain "why does a hash table give O(1) lookup?"

### Level 2 (Academic DSA) — The Scholar Trial
- **Trial 1**: implement Merge Sort, Dijkstra, DSU (with path compression), Segment Tree (basic) in <90 min.
- **Trial 2**: solve [LC 322 (Coin Change)](https://leetcode.com/problems/coin-change), [LC 207 (Course Schedule)](https://leetcode.com/problems/course-schedule), [LC 295 (Find Median)](https://leetcode.com/problems/find-median-from-data-stream).
- **Trial 3**: explain "why does DP need overlapping subproblems?"

### Level 3 (Placement DSA) — The Job-Ready Trial
- **Trial 1**: implement Lazy Segtree, Trie, LRU Cache in <90 min.
- **Trial 2**: solve [LC 76 (Min Window Substring)](https://leetcode.com/problems/minimum-window-substring), [LC 84 (Largest Rectangle)](https://leetcode.com/problems/largest-rectangle-in-histogram), [LC 124 (Max Path Sum)](https://leetcode.com/problems/binary-tree-maximum-path-sum).
- **Trial 3**: 25-min mock interview on a Medium with full narration.

### Level 4 (Product-Based DSA) — The Unicorn Trial
- **Trial 1**: implement Segment Tree (lazy), KMP, BIT, Sparse Table in <2 hours.
- **Trial 2**: solve [LC 218 (Skyline)](https://leetcode.com/problems/the-skyline-problem), [LC 312 (Burst Balloons)](https://leetcode.com/problems/burst-balloons), [LC 1192 (Critical Connections)](https://leetcode.com/problems/critical-connections-in-a-network).
- **Trial 3**: design + present LRU Cache + RateLimiter system.

### Level 5 (FAANG DSA) — The FAANG Trial
- **Trial 1**: in 5 hours, simulate a full FAANG onsite (4 coding + 1 system design).
- **Trial 2**: solve 5 random LC Hards from your weakest topic, ≥4 in 45 min each.
- **Trial 3**: walk through 10 prepared behavioral stories (STAR), 2-3 min each, polished.

### Level 6 (Competitive Programming) — The Codeforces Trial
- **Trial 1**: implement Lazy Segtree, BIT, Suffix Array (O(n log² n)), Number Theory toolkit, FFT in <4 hours.
- **Trial 2**: maintain Codeforces **Expert (1600+)** rating for 30 days.
- **Trial 3**: solve a Div 2 D in <40 min during a live contest.

### Level 7 (Elite CP) — The Grandmaster Trial
- **Trial 1**: implement HLD, Dinic's, Suffix Automaton, FFT, Treap in <8 hours.
- **Trial 2**: maintain CF **Master (2100+)** for 60 days.
- **Trial 3**: ICPC Regional finalist OR teach a 1-hour session on a hard CP topic.

### Level 8 (Algorithm Engineering) — The Architect Trial
- **Trial 1**: build a production-grade KV store / rate limiter / search engine.
- **Trial 2**: benchmark against existing libraries; document tradeoffs.
- **Trial 3**: present at a meetup / publish a technical blog post.

### Level 9 (Research-Level) — The Researcher Trial
- **Trial 1**: re-implement a state-of-the-art algorithm from a paper.
- **Trial 2**: identify an open problem; make a substantive observation.
- **Trial 3**: submit a paper / preprint / conference workshop.

### Level 10 (Algorithm Inventor) — The Final Trial
- **Trial 1**: invent a novel algorithm or improvement.
- **Trial 2**: publish at a top venue (SODA, STOC, ESA, ICALP, ...) OR open-source a widely-used implementation.
- **Trial 3**: train the next generation of grandmasters.

## 📋 THE WEEKLY MICRO-TRIAL

Every Sunday, run a 30-minute "spot check":

```
☐ Pick 3 random algorithms; implement from memory in <10 min each
☐ Pick 1 random LC Medium; solve in <25 min
☐ Pick 1 concept; teach it aloud (record yourself)
☐ Score: 1 = struggled; 5 = effortless. Goal: 4+ each.
```

If you can't pass weekly micro-trials, you've **leaked** mastery somewhere. Re-cement.

## 🔄 RE-VERIFICATION RHYTHM

Knowledge decays. Re-verify each level:

- Level 0-2: every 3 months (foundational; keep sharp)
- Level 3-5: every 6 months (interview-relevant; can rust)
- Level 6-7: continuously (CP keeps you sharp naturally)
- Level 8-10: ongoing (research/work IS the verification)
