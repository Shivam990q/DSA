# 🛠️ Tools & IDEs

> *"A craftsman's tools don't make the work — but bad tools slow it down."*

---

## EDITORS / IDEs
- **VS Code** ⭐ — universal, extensible, free
- **CLion** — JetBrains, CMake-based, great debugger
- **Vim / Neovim** — CP veterans, fast editing
- **Sublime Text** — lightweight, fast
- **Code::Blocks** — simple C++ IDE

## ONLINE IDEs / JUDGES
- **ideone.com** — quick testing
- **godbolt.org (Compiler Explorer)** — view assembly, optimization study ⭐
- **leetcode.com/playground**
- **codeforces.com/run** — custom test

## COMPILERS / INTERPRETERS
- **g++** (C++17/20) — `g++ -std=c++17 -O2 -Wall`
- **Python 3.11+** / **PyPy3** (PyPy for CP speed)
- **JDK 17+** (Java)

## DEBUGGING
- **gdb** (Linux), **lldb** (Mac)
- **Visual Studio Debugger**
- **Sanitizers** ⭐: `-fsanitize=address,undefined` (catches UB, out-of-bounds)
- **Valgrind** — memory leaks
- Debug macros (`#ifdef LOCAL`)

## CP-SPECIFIC TOOLS
- **Competitive Companion** (browser extension) — parses problems
- **cf-tool** (CLI for [Codeforces](https://codeforces.com))
- **Stress-test scripts** (see [`../19-TEMPLATES-AND-IMPLEMENTATIONS/stress-test-runner.sh`](../19-TEMPLATES-AND-IMPLEMENTATIONS/stress-test-runner.sh))
- **CF Predictor / carrot** (rating extensions)

## PROFILING (Level 8+)
- **perf** (Linux)
- **Intel VTune**
- **gprof / callgrind**
- **Chrome DevTools** (for JS)

## NOTE-TAKING / JOURNALS
- **Obsidian** ⭐ — markdown, linked notes (great for pattern journal)
- **Notion** — flexible
- **Anki** — spaced repetition flashcards
- **Plain markdown files** — simplest, portable

---

## RECOMMENDED CP SETUP
```
1. VS Code or CLion
2. g++ with -O2 -std=c++17 -fsanitize=address,undefined (local)
3. Competitive Companion extension
4. Stress-test scripts ready
5. Template file + snippets
6. Obsidian for pattern journal + mistake log
```

---

## THE PRINCIPLE
Set up ONCE, well. Remove friction between intent and action. Then forget the tools and focus on the work.
