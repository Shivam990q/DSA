# 📦 Code Libraries (CP)

> *"Tested, battle-hardened implementations. Study them, adapt them, never copy blindly."*

---

## THE TOP LIBRARIES
- **[KACTL](https://github.com/kth-competitive-programming/kactl)** ⭐ (KTH Algorithm Competition Template Library) — compact, ICPC-ready, the gold standard
- **[AtCoder Library (ACL)](https://github.com/atcoder/ac-library)** — official, well-tested (segtree, lazy segtree, DSU, max flow, MCMF, FFT, math)
- **Benq's library** — comprehensive, well-organized
- **EbTech's library** (Rust) — well-documented
- **Bjarki Ágúst Guðmundsson's library**
- **kactl-style personal libraries** (many top competitors publish theirs)

---

## WHAT THESE LIBRARIES CONTAIN
- Data structures: segment tree, lazy segtree, BIT, sparse table, treap, link-cut tree
- Graph: Dijkstra, max flow (Dinic), MCMF, matching, SCC, bridges, LCA, HLD
- Strings: KMP, Z, suffix array, suffix automaton, Aho-Corasick, hashing
- Math: modular arithmetic, sieve, Miller-Rabin, Pollard rho, CRT, FFT/NTT, combinatorics
- Geometry: convex hull, line intersection, point-in-polygon, closest pair

---

## HOW TO USE A LIBRARY
1. **Study the implementation** — understand HOW it works
2. **Adapt to your style** — rename, simplify, comment
3. **Test it yourself** — stress-test every component
4. **Memorize the interface** — know how to call it under pressure
5. **NEVER copy blindly** — a library you don't understand is a liability

---

## BUILD YOUR OWN
The best library is one you built and understand:
- Start with the basics (DSU, segment tree, BIT)
- Add as you learn (each new algorithm → tested template)
- Organize (one file per structure, or a snippets manager)
- See [`../19-TEMPLATES-AND-IMPLEMENTATIONS/`](../19-TEMPLATES-AND-IMPLEMENTATIONS/) for starter implementations

---

## THE ICPC NOTEBOOK
For ICPC, teams compile a printed reference (25 pages max). KACTL is designed exactly for this — compact, tested, printable. Build your team's notebook from KACTL + your own additions.

---

## THE WARNING
A library accelerates IMPLEMENTATION, not THINKING. The observation, algorithm choice, and proof are still yours. Don't let a library become a crutch that lets you skip understanding.
