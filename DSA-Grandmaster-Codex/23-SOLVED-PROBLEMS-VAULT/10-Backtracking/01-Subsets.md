# Subsets

**Platform**: LeetCode 78 · **Difficulty**: Medium · **Topics**: Array, Backtracking, Bit Manipulation · **Pattern**: Power set / choose-or-skip

---

## 📜 Problem Statement

Given an integer array `nums` of **unique** elements, return *all possible subsets (the power set)*.

The solution set **must not** contain duplicate subsets. Return the solution in **any order**.

### Examples

**Example 1:**
```
Input:  nums = [1, 2, 3]
Output: [[], [1], [2], [1,2], [3], [1,3], [2,3], [1,2,3]]
Explanation: There are 2^3 = 8 subsets, including the empty set.
```

**Example 2:**
```
Input:  nums = [0]
Output: [[], [0]]
```

**Example 3:**
```
Input:  nums = [9, 8]
Output: [[], [9], [8], [9,8]]
```

### Constraints
```
1 <= nums.length <= 10
-10 <= nums[i] <= 10
All the numbers of nums are unique.
```

---

## 🧠 Understanding the problem

A **subset** is any selection of elements where each element is either **in** or **out** — and *order does not matter*, so `{1,2}` and `{2,1}` are the same subset and must appear only once. With `n` distinct elements, every element makes one independent binary choice (in/out), so there are exactly `2^n` subsets, including the empty set and the full set.

Because the elements are **unique**, we never have to worry about duplicate subsets arising from equal values (that headache is reserved for [Subsets II](./05-Subsets-II.md)). Our only job is to enumerate the `2^n` selections systematically without missing any and without repeating any.

The constraint `n <= 10` means `2^n <= 1024` — tiny. So we don't need cleverness for speed; we need a clean enumeration scheme that is obviously complete and duplicate-free.

## Approach 1 — Backtracking with a start index ⭐

### Intuition
Build subsets by walking left to right and, at each position, deciding whether to extend the current path with the next element. To guarantee *order doesn't matter* (so we never produce both `[1,2]` and `[2,1]`), every recursive call only looks **forward** from a `start` index. Each node of the recursion tree *is itself a valid subset*, so we record the path at **every node**, not just at leaves.

This is the canonical "combinations" skeleton: move forward with a `start` index.

### Algorithm
1. Maintain a growing path `cur` and the result list `res`.
2. On entering `backtrack(start)`: record a **copy** of `cur` (it's a valid subset).
3. For each `i` from `start` to `n-1`:
   - **choose**: append `nums[i]` to `cur`.
   - **explore**: recurse with `backtrack(i + 1)` (forward only).
   - **un-choose**: pop the last element of `cur`.

### Dry run on `[1, 2, 3]`
```
bt(0) cur=[]            record []
 i=0 push1 → bt(1) cur=[1]      record [1]
   i=1 push2 → bt(2) cur=[1,2]    record [1,2]
     i=2 push3 → bt(3) cur=[1,2,3]  record [1,2,3]
     pop3
   pop2
   i=2 push3 → bt(3) cur=[1,3]    record [1,3]
   pop3
 pop1
 i=1 push2 → bt(2) cur=[2]      record [2]
   i=2 push3 → bt(3) cur=[2,3]    record [2,3]
   pop3
 pop2
 i=2 push3 → bt(3) cur=[3]      record [3]
 pop3
Result: [], [1], [1,2], [1,2,3], [1,3], [2], [2,3], [3]
```
Every element only ever recurses to the right, so each subset is generated by exactly one path.

### Code
```cpp
class Solution {
public:
    void backtrack(int start, vector<int>& nums, vector<int>& cur,
                   vector<vector<int>>& res) {
        res.push_back(cur);                 // every node is a valid subset
        for (int i = start; i < (int)nums.size(); i++) {
            cur.push_back(nums[i]);         // choose
            backtrack(i + 1, nums, cur, res); // explore (forward only)
            cur.pop_back();                 // un-choose
        }
    }
    vector<vector<int>> subsets(vector<int>& nums) {
        vector<vector<int>> res;
        vector<int> cur;
        backtrack(0, nums, cur, res);
        return res;
    }
};
```
```java
class Solution {
    public List<List<Integer>> subsets(int[] nums) {
        List<List<Integer>> res = new ArrayList<>();
        backtrack(0, nums, new ArrayList<>(), res);
        return res;
    }
    private void backtrack(int start, int[] nums, List<Integer> cur,
                           List<List<Integer>> res) {
        res.add(new ArrayList<>(cur));            // every node is a valid subset
        for (int i = start; i < nums.length; i++) {
            cur.add(nums[i]);                     // choose
            backtrack(i + 1, nums, cur, res);     // explore (forward only)
            cur.remove(cur.size() - 1);           // un-choose
        }
    }
}
```
```python
class Solution:
    def subsets(self, nums):
        res = []
        def backtrack(start, cur):
            res.append(cur[:])                    # every node is a valid subset
            for i in range(start, len(nums)):
                cur.append(nums[i])               # choose
                backtrack(i + 1, cur)             # explore (forward only)
                cur.pop()                         # un-choose
        backtrack(0, [])
        return res
```

### Complexity
- **Time**: O(n · 2^n) — there are `2^n` subsets and copying each into the result costs up to O(n).
- **Space**: O(n) auxiliary for the recursion depth and the `cur` path (output not counted).

## Approach 2 — Iterative "grow the result"

### Intuition
Start with the power set of nothing — `[[]]` — and fold in one element at a time. When you introduce a new element `x`, every subset built so far spawns a new subset that is *itself plus `x`*. This doubles the count each round, matching `2^n`.

### Algorithm
1. Initialize `res = [[]]`.
2. For each `x` in `nums`:
   - For every existing subset `s` in `res`, create `s + [x]` and append it to `res`.
3. Return `res`.

### Dry run on `[1, 2, 3]`
```
start                 res = [[]]
add 1 → append []+1   res = [[], [1]]
add 2 → append [],[1]+2  res = [[], [1], [2], [1,2]]
add 3 → append all +3 res = [[], [1], [2], [1,2], [3], [1,3], [2,3], [1,2,3]]
```

### Code
```cpp
class Solution {
public:
    vector<vector<int>> subsets(vector<int>& nums) {
        vector<vector<int>> res = {{}};
        for (int x : nums) {
            int sz = res.size();
            for (int i = 0; i < sz; i++) {
                vector<int> s = res[i];
                s.push_back(x);
                res.push_back(s);
            }
        }
        return res;
    }
};
```
```java
class Solution {
    public List<List<Integer>> subsets(int[] nums) {
        List<List<Integer>> res = new ArrayList<>();
        res.add(new ArrayList<>());
        for (int x : nums) {
            int sz = res.size();
            for (int i = 0; i < sz; i++) {
                List<Integer> s = new ArrayList<>(res.get(i));
                s.add(x);
                res.add(s);
            }
        }
        return res;
    }
}
```
```python
class Solution:
    def subsets(self, nums):
        res = [[]]
        for x in nums:
            res += [s + [x] for s in res]
        return res
```

### Complexity
- **Time**: O(n · 2^n) — same total work building `2^n` subsets.
- **Space**: O(1) auxiliary (output not counted; no recursion stack).

## Approach 3 — Bitmask enumeration

### Intuition
Each subset corresponds one-to-one with an `n`-bit number from `0` to `2^n - 1`: bit `j` set means "include `nums[j]`." Iterate over all bitmasks and decode each into a subset. This makes the `2^n` count literally visible.

### Algorithm
1. For `mask` from `0` to `2^n - 1`:
   - Build a subset: for each bit `j`, if `mask` has bit `j` set, include `nums[j]`.
   - Append it to `res`.

### Dry run on `[1, 2, 3]`
```
mask 000 → []        mask 100 → [3]
mask 001 → [1]       mask 101 → [1,3]
mask 010 → [2]       mask 110 → [2,3]
mask 011 → [1,2]     mask 111 → [1,2,3]
```

### Code
```cpp
class Solution {
public:
    vector<vector<int>> subsets(vector<int>& nums) {
        int n = nums.size();
        vector<vector<int>> res;
        for (int mask = 0; mask < (1 << n); mask++) {
            vector<int> cur;
            for (int j = 0; j < n; j++)
                if (mask & (1 << j)) cur.push_back(nums[j]);
            res.push_back(cur);
        }
        return res;
    }
};
```
```java
class Solution {
    public List<List<Integer>> subsets(int[] nums) {
        int n = nums.length;
        List<List<Integer>> res = new ArrayList<>();
        for (int mask = 0; mask < (1 << n); mask++) {
            List<Integer> cur = new ArrayList<>();
            for (int j = 0; j < n; j++)
                if ((mask & (1 << j)) != 0) cur.add(nums[j]);
            res.add(cur);
        }
        return res;
    }
}
```
```python
class Solution:
    def subsets(self, nums):
        n = len(nums)
        res = []
        for mask in range(1 << n):
            cur = [nums[j] for j in range(n) if mask & (1 << j)]
            res.append(cur)
        return res
```

### Complexity
- **Time**: O(n · 2^n) — `2^n` masks, each decoded in O(n).
- **Space**: O(1) auxiliary (output not counted).

## ⚖️ Approach comparison

| Approach | Time | Space | When to mention |
|----------|------|-------|-----------------|
| Backtracking + start index | O(n·2^n) | O(n) | the default; extends to dup/constraint variants ⭐ |
| Iterative grow | O(n·2^n) | O(1) | no recursion; clean and intuitive |
| Bitmask | O(n·2^n) | O(1) | elegant when `n` is small (≤ ~20) |

All three are optimal in total work (you cannot beat `2^n` outputs). Backtracking is the one to master because it generalizes directly to [Subsets II](./05-Subsets-II.md), [Combination Sum](./02-Combination-Sum.md), and partitioning problems.

## 🧪 Edge cases & pitfalls
- **Empty input** is outside the constraints (`n >= 1`), but every approach naturally returns `[[]]` for it.
- **Single element** `[0]` → `[[], [0]]`. The empty set is always part of the answer; don't forget to record it (in the backtracking version it's recorded at the very first node).
- **Pitfall — record a copy**: appending `cur` by reference (instead of a copy) means later `pop_back`s corrupt subsets already stored. Always push `new ArrayList<>(cur)` / `cur[:]` / a value copy.
- **Pitfall — wrong forward index**: recursing with `start` (or `0`) instead of `i + 1` produces duplicates and infinite-style repeats. For unique-element subsets you must advance to `i + 1`.
- **Bitmask overflow**: `1 << n` is fine for `n <= 10`; for larger `n` use `1L << n` in C++/Java.
