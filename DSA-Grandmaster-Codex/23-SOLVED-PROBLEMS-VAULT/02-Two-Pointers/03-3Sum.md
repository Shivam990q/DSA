# 3Sum

**Platform**: LeetCode 15 · **Difficulty**: Medium · **Topics**: Array, Two Pointers, Sorting · **Pattern**: Fix one + two pointers

---

## 📜 Problem Statement

Given an integer array `nums`, return all the triplets `[nums[i], nums[j], nums[k]]` such that `i != j`, `i != k`, and `j != k`, and `nums[i] + nums[j] + nums[k] == 0`.

The solution set must not contain **duplicate triplets**.

### Examples

**Example 1:**
```
Input:  nums = [-1,0,1,2,-1,-4]
Output: [[-1,-1,2],[-1,0,1]]
Explanation: The distinct triplets summing to 0 are [-1,-1,2] and [-1,0,1].
```

**Example 2:**
```
Input:  nums = [0,1,1]
Output: []
```

**Example 3:**
```
Input:  nums = [0,0,0]
Output: [[0,0,0]]
```

### Constraints
```
3 <= nums.length <= 3000
-10^5 <= nums[i] <= 10^5
```

---

## 🧠 Understanding the problem

Two challenges: (1) find triplets summing to 0, (2) avoid duplicate triplets. Sorting solves both — it enables two pointers AND makes duplicate-skipping trivial (skip consecutive equal values at the same "level").

---

## Approach 1 — Brute force (all triples)

### Intuition
Three nested loops. Use a set to deduplicate.

### Complexity
- **Time**: O(n³). **Space**: O(n) for dedup set.

### Verdict
Too slow for n=3000 (2.7×10¹⁰ ops). Baseline only.

---

## Approach 2 — Sort + hash set for pair

### Intuition
Sort. Fix `i`. For each `i`, use a hash set to find pairs `(j, k)` summing to `-nums[i]`.

### Complexity
- **Time**: O(n²). **Space**: O(n) for the set.

### Verdict
Works, but two pointers is cleaner and O(1) extra space.

---

## Approach 3 — Sort + two pointers (optimal) ⭐

### Intuition
Sort the array. Fix `i` (the first element). For the remaining `[i+1, n-1]`, use two pointers `l=i+1, r=n-1` to find pairs summing to `-nums[i]`. Skip duplicates at both the `i` level and the `l/r` level.

### Algorithm
1. Sort `nums`.
2. For `i` from 0 to n-3:
   - If `nums[i] > 0`: break (no way to sum to 0 with all positives).
   - If `i > 0 && nums[i] == nums[i-1]`: skip (duplicate first element).
   - `l = i+1, r = n-1`.
   - While `l < r`:
     - `sum = nums[i] + nums[l] + nums[r]`.
     - If `sum < 0`: `l++`.
     - If `sum > 0`: `r--`.
     - If `sum == 0`: record `[nums[i], nums[l], nums[r]]`; skip duplicate `l`s and `r`s; `l++, r--`.

### Dry run on `[-1,0,1,2,-1,-4]`
```
sorted: [-4,-1,-1,0,1,2]
i=0 (val=-4): l=1,r=5 → sum=-4+(-1)+2=-3<0 → l++ → ... no triplet
i=1 (val=-1): l=2,r=5 → sum=-1+(-1)+2=0 → record [-1,-1,2]; skip dups; l++,r--
              l=3,r=4 → sum=-1+0+1=0 → record [-1,0,1]
i=2 (val=-1): skip (same as i=1)
i=3 (val=0): l=4,r=5 → sum=0+1+2=3>0 → r-- → l>=r → done
Result: [[-1,-1,2],[-1,0,1]]
```

### Code
```cpp
vector<vector<int>> threeSum(vector<int>& nums) {
    sort(nums.begin(), nums.end());
    int n = nums.size();
    vector<vector<int>> res;
    for (int i = 0; i < n - 2; i++) {
        if (nums[i] > 0) break;
        if (i > 0 && nums[i] == nums[i-1]) continue;
        int l = i + 1, r = n - 1;
        while (l < r) {
            int sum = nums[i] + nums[l] + nums[r];
            if (sum < 0) l++;
            else if (sum > 0) r--;
            else {
                res.push_back({nums[i], nums[l], nums[r]});
                l++; r--;
                while (l < r && nums[l] == nums[l-1]) l++;
                while (l < r && nums[r] == nums[r+1]) r--;
            }
        }
    }
    return res;
}
```

```python
def threeSum(nums):
    nums.sort()
    n, res = len(nums), []
    for i in range(n - 2):
        if nums[i] > 0:
            break
        if i > 0 and nums[i] == nums[i-1]:
            continue
        l, r = i + 1, n - 1
        while l < r:
            s = nums[i] + nums[l] + nums[r]
            if s < 0:
                l += 1
            elif s > 0:
                r -= 1
            else:
                res.append([nums[i], nums[l], nums[r]])
                l += 1; r -= 1
                while l < r and nums[l] == nums[l-1]:
                    l += 1
                while l < r and nums[r] == nums[r+1]:
                    r -= 1
    return res
```

### Complexity
- **Time**: O(n²) — outer loop O(n) × inner two-pointer O(n).
- **Space**: O(1) extra (ignoring output and sort).

### Verdict
**The optimal answer.** O(n²) is the best possible (proven lower bound for 3SUM in comparison model). Clean duplicate handling via sorting + skip.

---

## ⚖️ Approach comparison

| Approach | Time | Space | Dedup method |
|----------|------|-------|--------------|
| Brute + set | O(n³) | O(n) | hash set of sorted triples |
| Sort + hash pair | O(n²) | O(n) | hash set |
| Sort + two pointers | **O(n²)** | **O(1)** | skip consecutive duplicates ⭐ |

---

## 🧪 Edge cases & pitfalls
- **All zeros** `[0,0,0,0]` → one triplet `[0,0,0]`. The skip logic handles it (first `i` finds it, rest skip).
- **No valid triplet** → empty result.
- **Fewer than 3 elements** → impossible per constraints, but guard if needed.
- **Pitfall**: skipping duplicates with wrong condition (`i > 0` not `i > start`; `nums[l] == nums[l-1]` not `nums[l] == nums[l+1]` — the direction matters).
- **Pitfall**: forgetting `nums[i] > 0` early break (optimization, not correctness — but speeds up).
