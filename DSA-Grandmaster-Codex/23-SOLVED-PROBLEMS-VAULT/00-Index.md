This is the missing half of the Codex. Everywhere else teaches concepts and points you to problems. **Here, every problem is a complete editorial.**

## 📐 THE EDITORIAL FORMAT (every problem)

Each problem lives in its **own file** (platform-style) and contains:

```
Problem Statement   : full description + constraints + multiple examples + follow-up
Understanding       : what the problem is really asking, key observations
Approach 1 (brute)  : intuition → algorithm → dry run → C++, Java & Python → complexity → verdict
Approach 2 (better) : same structure, improving on the last
Approach 3 (optimal): same structure, the answer you'd present
Approach 4+ (alt)   : other valid angles (sort / heap / union-find / bitmask / quickselect ...)
Approach comparison : a table of time/space/trade-offs
Edge cases & pitfalls
Related problems
```

> **Every approach gives complete, runnable code in all three interview languages — C++, Java, and Python.**

Every problem is approached **multiple ways** — brute force first (to prove understanding), then each optimization with the *reasoning* for why it's better, so you learn to DERIVE the optimal, not memorize it.

**How to use it**: open a problem, read ONLY the statement, attempt it cold (30–45 min). Then read the approaches in order, feeling WHY each improves on the last. Re-derive the optimal from the key insight after 7 and 30 days.
